Template Inheritance

  • enables the creation of reusable and modular web layouts by establishing a parent-child relationship between templates
  • allows you to create a base layout (a "skeleton") and then define child templates that inherit from this base and override specific sections (called "blocks")
  • ideal for maintaining a consistent structure across multiple pages while allowing for content variations
  • templates can extend other child templates, creating a hierarchy of inheritance for complex layouts
  • like object-oriented programming inheritance, where a child class inherits properties and methods from a parent class and can override or add its own

Base Template (Parent)

  • defines the common structure and elements shared across multiple pages, such as headers, footers, navigation bars, and sidebars
  • utilizes {% block %} tags to define named areas where child templates can insert or override content

Child Template (Extending)

  • inherit from a base template using the {% extends %} tag
  • then use {% block %} tags with the same names as those in the parent to inject their specific content, overriding or extending the parent's default content
  • {{ super() }} can be used to render the content of the parent block, allowing for appending or prepending content without completely overriding it
